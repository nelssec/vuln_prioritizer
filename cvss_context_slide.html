Why You Need More Than Just CVSS Score

Understanding CVSS Limitations & The Need for Contextual Vulnerability Management

CVSS 3.1 Current Standard

8 Base Metrics:

AV Attack Vector
AC Attack Complexity
PR Privileges Required
UI User Interaction
S Scope
C Confidentiality
I Integrity
A Availability
Version Inconsistency Issue:
Same CVE can score differently - Example: 5.5 (Medium) in v3 but 2.1 (Low) in v2

CVSS 4.0 Newer

Key Improvements:

  • Supplemental metrics for better context
  • Automatable & Recovery metrics
  • Provider Urgency indicator
  • Exploit Maturity refinements
  • Safety impact considerations

Still Missing: Business context, asset criticality, threat intelligence integration

What CVSS Gives You

  • Standardized Scoring: Consistent 0-10 scale across all vulnerabilities
  • Technical Severity: Assessment based on exploit mechanics & impact
  • Common Language: Critical/High/Medium/Low rankings
  • Vendor-Agnostic: Compare vulnerabilities across different products
  • Repeatable: Same inputs = same score
  • Base Metrics: Attack vector, complexity, privileges, CIA impact

What You Actually Need

  • Business Context: Is this our crown jewel system or test environment?
  • Asset Criticality: Customer-facing? Revenue-generating? Compliance-scoped?
  • Threat Intelligence: Is this being actively exploited in the wild?
  • Environmental Factors: Do we have compensating controls?
  • Exposure Assessment: Internet-facing or internal only?
  • Data Sensitivity: What type of data could be compromised?
  • Exploit Availability: Is there public exploit code available?
  • Patch Complexity: Can we actually fix this quickly?

Key Takeaway

CVSS is "only one piece of a much larger vulnerability management puzzle" - it provides valuable technical baseline scoring, but effective prioritization requires layering in threat intelligence, asset criticality, business impact, and environmental context to make informed remediation decisions.
